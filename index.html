Don't Archive, Transfer

Why Archiving is Harmful

Archiving a project might seem like a clean exit, but it can cause significant problems for the community:

  • Issues are lost: All unresolved issues, suggestions, and discussions vanish, cutting off valuable context for future development.
  • Partial history loss: Forks often lack the full context or connections to the original repository.
  • PyPI namespace loss: When the repository is archived, the PyPI namespace becomes inaccessible, blocking further development on the package name.

The Better Alternative: Transfer Ownership

If you can no longer maintain a project, consider transferring it instead of archiving. Here's why:

  • Continuity: The project remains active and accessible to users and contributors.
  • Preservation: Issues, discussions, and context are retained.
  • Namespace Integrity: PyPI namespaces and repository URLs remain intact, avoiding disruptions.

How to Transfer a Project

  1. Find a trusted maintainer: Look for active contributors or community members willing to take over.
  2. Transfer repository ownership: Use GitHub's repository transfer feature to hand over the project.
  3. Transfer PyPI access: Add the new maintainer to your PyPI project via the PyPI role management interface.
  4. Announce the transition: Inform users and contributors about the new maintainer and future plans.

Tools to Help

Consider using these tools to facilitate the process:

  • GitHub's Repository Transfer
  • PyPI Role Management
  • OpenFaaS for community-driven maintenance models
